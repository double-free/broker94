119 /**
120  * OES异步API的连接通道运行时信息
121  *
122  * <p> @note 关于上一次会话的最大请求数据编号 (针对委托通道)
123  * - 将通过 lastOutMsgSeq 字段存储上一次会话实际已发送的出向消息序号, 即: 服务器端最
124  * 后接收到的客户委托流水号(clSeqNo)
125  * - 该值对应的是服务器端最后接收到并校验通过的(同一环境号下的)客户委托流水号, 效果等价
126  * 于 OesApi_InitOrdChannel接口的pLastClSeqNo参数的输出值
127  * - 登录成功以后, API会将该值存储在 <code>pAsyncChannel->lastOutMsgSeq</code>
128  * 和 <code>pSessionInfo->lastOutMsgSeq</code> 字段中
129  * - 该字段在登录成功以后就不会再更新
130  * - 客户端若想借用这个字段来维护自增的"客户委托流水号(clSeqNo)"也是可行的, 只是需要注
131  * 意该字段在登录后会被重置为服务器端最后接收到并校验通过的"客户委托流水号(clSeqNo)"
132  * </p>
133  *
134  * <p> @note 关于最近接收到的回报数据编号 (针对回报通道):
135  * - 将通过 lastInMsgSeq 字段存储上一次会话实际接收到的入向消息序号, 即: 最近接收到的
136  * 回报数据编号
137  * - 该字段会在接收到回报数据并回调OnMsg成功以后实时更新, 可以通过该字段获取到最近接收到
138  * 的回报数据编号
139  * - 当OnConnect函数指针为空时, 会执行默认的回报订阅处理, 此时会自动从断点位置继续订阅
140  * 回报数据
141  * - 若指定了OnConnect回调函数(函数指针不为空), 则需要显式的执行回报订阅处理
142  * - API会将回报数据的断点位置存储在 <code>pAsyncChannel->lastInMsgSeq</code>
143  * 和 <code>pSessionInfo->lastInMsgSeq</code> 字段中, 可以使用该值订阅回报
144  * - 客户端可以在OnConnect回调函数中重新设置
145  * <code>pSessionInfo->lastInMsgSeq</code> 的取值来重新指定初始的回报订阅位置,
146  * 效果等同于OesApi_InitRptChannel接口的lastRptSeqNum参数:
147  * - 等于0, 从头开始推送回报数据 (默认值)
148  * - 大于0, 以指定的回报编号为起点, 从该回报编号的下一条数据开始推送
149  * - 小于0, 从最新的数据开始推送回报数据
150  * </p>
151  */
152 typedef SEndpointChannelT OesAsyncApiChannelT;

227 /**
228  * 异步API线程连接或重新连接完成后的回调函数的函数原型定义
229  *
230  * <p> 回调函数说明:
231  * - 对于回报通道, 需要通过该回调函数完成回报订阅操作。若函数指针为空, 则会使用通道配置中默认的

236  * <p> 关于回报数据的断点恢复 (针对回报通道):
237  * - 当OnConnect函数指针为空时, 会执行默认的回报订阅处理, 此时会自动从断点位置继续订阅回报数据

239  * - API会将回报数据的断点位置存储在 <code>pAsyncChannel->lastInMsgSeq</code> 和
240  * <code>pSessionInfo->lastInMsgSeq</code> 字段中, 可以使用该值订阅回报
241  * </p>
242  *
243  * <p> 线程说明:
244  * - OnMsg/OnConnect/OnDisconnect 回调函数均运行在异步API线程下
245  * </p>
246  *
247  * @param pAsyncChannel 异步API的连接通道信息
248  * @param pCallbackParams 外部传入的参数
249  * @retval =0 等于0, 成功
250  * @retval >0 大于0, 处理失败, 将重建连接并继续尝试执行
251  * @retval <0 小于0, 处理失败, 异步线程将中止运行
